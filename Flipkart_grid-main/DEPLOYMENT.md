# ParkWatch AI — Deployment Blueprint

Target architecture:

```
[ Vercel ]            [ Render ]                 [ Supabase ]      [ optional ]
React/Vite  ──HTTPS──▶ FastAPI (api.py)  ──SQL──▶ PostgreSQL       Chroma (RAG)
 frontend             JWT · agent · ETL          violations/logs   Groq (LLM)
```

Every cloud dependency is **optional** — the backend boots and serves the full
demo (in-memory data, keyword RAG, seasonal forecasts, deterministic agent)
with zero external services. Add credentials to progressively light up
persistence, vector RAG, real LLM, and ML forecasting.

---

## 1. Database — Supabase (or Neon) PostgreSQL

1. Create a project → copy the connection string (Settings → Database → URI).
2. Run the schema:
   ```bash
   psql "postgresql://USER:PASS@HOST:5432/postgres" -f Flipkart_grid-main/backend/database/schema.sql
   ```
   (Or paste `backend/database/schema.sql` into the Supabase SQL editor.)
3. The backend auto-creates tables on startup too, if `DATABASE_URL` is set and
   `SQLAlchemy` + `psycopg2-binary` are installed.

## 2. Backend — Render

- Blueprint: [`render.yaml`](render.yaml) (rootDir → `Flipkart_grid-main/backend`).
- Build: `pip install -r requirements.txt` · Start: `uvicorn api.main:app --host 0.0.0.0 --port $PORT`.
- To enable the production stack, uncomment the optional block in
  `backend/requirements.txt` (SQLAlchemy, psycopg2, chromadb, sentence-transformers,
  prophet/xgboost, langchain/langgraph, rapidfuzz).
- Env vars (Render dashboard):
  | Key | Purpose |
  |-----|---------|
  | `DATABASE_URL` | Supabase/Neon URI → persistence + agent memory |
  | `GROQ_API_KEY` | Real LLM replies (else deterministic analysis) |
  | `CORS_ORIGINS` | Your Vercel domain, e.g. `https://parkwatch.vercel.app` |
  | `PARKWATCH_SECRET` | JWT signing (auto-generated by the blueprint) |
- Health check: `GET /api/health` → `{ status, db, rag }`.

## 3. Frontend — Vercel

- Blueprint: [`frontend/vercel.json`](frontend/vercel.json) (framework `vite`, SPA rewrite).
- Root directory: `Flipkart_grid-main/frontend`.
- Env var: `VITE_API_BASE = https://<your-render-app>.onrender.com/api`
  (consumed in `src/lib/api.js`).
- Deploy → Vercel runs `npm run build`, serves `dist/`.

## 4. Local development

```bash
# backend
cd Flipkart_grid-main/backend
pip install -r requirements.txt
cp .env.example .env          # add GROQ_API_KEY / DATABASE_URL if you have them
uvicorn api.main:app --reload --port 8000

# frontend
cd Flipkart_grid-main/frontend
npm install
npm run dev                   # http://localhost:5173
```

## Graceful-degradation matrix

| Capability | Without extras | With extras |
|---|---|---|
| Data store | in-memory demo frame | PostgreSQL (`DATABASE_URL`) |
| Agent memory | per-request history | last 10 turns from `interaction_logs` |
| RAG | keyword overlap | ChromaDB + MiniLM embeddings |
| LLM replies | deterministic explainable analysis | Groq Llama-3.3-70B (`GROQ_API_KEY`) |
| Forecasting | seasonal (day×hour) | Prophet / XGBoost |
| ETL fuzzy headers | stdlib difflib | rapidfuzz |
| Fleet VRP | nearest-neighbour | OR-Tools CVRP (`ortools`) |
